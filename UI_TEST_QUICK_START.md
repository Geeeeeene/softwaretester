# UI测试快速开始 🚀

## 5分钟快速上手

### 1️⃣ 创建UI测试项目

在前端"项目管理"页面点击"创建项目"：

- **项目名称**：我的UI测试
- **项目类型**：选择"UI测试"
- **源代码路径**：`C:\path\to\your\app.exe`

### 2️⃣ 准备图像资源

将UI元素截图（PNG格式）放到：
```
backend/examples/robot_resources/
```

例如：
- `main_window.png` - 主窗口
- `button.png` - 按钮

### 3️⃣ 生成测试用例

点击"UI测试（使用AI生成UI测试用例）"：

**测试用例名称**：应用启动测试

**测试描述**：
```
测试应用程序能否正常启动并显示主窗口。
测试步骤：
1. 启动应用程序
2. 等待5秒让应用完全加载
3. 验证主窗口是否出现（检查main_window.png）
4. 关闭应用程序
```

点击"生成测试用例"，AI会自动生成Robot Framework脚本！

### 4️⃣ 执行测试

查看生成的脚本，确认无误后点击"执行UI测试"。

### 5️⃣ 查看结果

测试完成后会显示：
- ✓ 测试通过 / ✗ 测试失败
- 执行时长
- 详细日志
- 生成的报告文件

## ⚙️ 环境要求

确保已配置：
- ✅ Python 3.8+
- ✅ Robot Framework
- ✅ SikuliLibrary
- ✅ Java 8+
- ✅ Claude API Key（在`backend/app/core/config.py`中配置）

## 🎯 核心特性

- 🤖 **AI驱动**：Claude自动生成测试脚本
- 🖼️ **图像识别**：基于SikuliLibrary，无需修改应用代码
- 📊 **完整统计**：测试历史、通过率、执行记录
- 🎨 **友好界面**：四栏式布局，操作简单直观

## 💡 提示

1. **测试描述要详细**：步骤越清晰，AI生成的脚本越准确
2. **图像要清晰**：使用高质量的PNG截图
3. **相似度设置**：默认0.7，可根据需要调整
4. **等待时间**：应用启动后要有足够的等待时间

开始您的UI自动化测试之旅吧！🎉
